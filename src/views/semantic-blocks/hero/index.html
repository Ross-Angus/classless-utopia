Hero panels

The width of the hero panel is determined by the container. In order to have a wider hero panel, it should sit in different positions in the markup.

The CSS for the hero panel assumes that a grid layout is used. Any standard grid can be used but only panels with content inside them will render with the background which allows text over an image to have sufficient contrast.

Empty elements can be used in order to create gaps for users to see the image behind the text.

<header>
  <picture>
    <!-- source tags, if required -->
    <img src="hero-image-name.jpg" alt="Hero image alt text">
  </picture>
  <div>
    <div>
      <h2>Hero heading</h2>
      <p>Hero intro text - assuming the parent `div` has two siblings,
      this will take up one third of the space on desktop.</p>
      <p><a href="#" rel="related">Call-to-action link</a></p>
    </div>
    <!-- these empty elements are used to create space -->
    <div></div>
    <div></div>
  </div>
</header>
