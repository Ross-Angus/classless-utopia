Classless Utopia - Main navigation

Main navigation

The main navigation can take a couple of different forms.

Collapsing

To create an infinitely nested collapsing navigation tree, use the following pattern:

<header>
  <nav>
    <ul>
      <li><a href="#">Top level item</a>
        <ul>
          <li><a href="#">Second level item</a></li>
        </ul>
      </li>
    </ul>
  </nav>
</header>

You can continue nesting lists as deep as you like (within reason).

The functionality and features of the main navigation drew inspiration from work by NN/g and ARIA Authoring Practices Guide. This leads to a lot of JavaScript concerned with labelling elements correctly, reflecting the navigation state back to the user and handling keyboard navigation.

Only the top level of navigation will display. Users will see a chevron inside a button to the right of each navigation item which has children. Clicking on the link will navigate to the page as normal. Clicking on the button will toggle the child nodes open and closed.

The navigation will display horizontally until the top line wraps. After this point, it will switch to a hamburger menu and vertical orientation. This means that there is no breakpoint perse - this switch is determined by the content of the navigation.

Accessibility markup is automatically added and updated as the user interacts with the navigation.

Linear

If a single row of navigation is required, the following pattern can be used:

<nav>
  <p>
    <a href="#">Top level navigation</a>
    <a href="#">Another top level navigation item</a>
    <a href="#">A third example</a>
  </p>
</nav>
