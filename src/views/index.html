Classless Utopia

This is an experiment in semantic HTML in order to see how far we can go to create a set of web components without classes or IDs. It's purely an academic exercise which has a few secret goals:

  • To demonstrate some little-used features of HTML and HTML5
  • To gently encourage a more semantic approach to HTML rather than div soup
  • To demonstrate some native browser functionality which is often replicated in JavaScript, harming site performance
  • To give me a grab-bag of useful techniques to use in the future
  • To force me to learn more about typography thus giving me more empathy with my designer siblings (this stretch goal was revealed to me along the way)

Rules

  1. It should be possible to create all of the components without the use of the class attribute
    1. Exception: I've used highlight.js to make example markup easier to read. This adds classes using JavaScript which is styled with its own CSS. This would not be required in a brochureware site created using this framework.
  2. No use of the ID attribute, for the purposes of styling elements
    1. Exception: The ID attribute can still be used on form elements etc
  3. It should be possible to create all of the components without the use of the data- attribute
    1. Exception: data-js attributes were used to add functionality and accessibility to the multi-level navigation component. These attributes were added to elements injected directly into the DOM by the JavaScript and were not intended to be used by whoever adds markup. These same attributes were reused for styling purposes.

What is allowed

  • Use of rel attributes, where such attributes have established semantic meaning (see the variety of call-to-action buttons and form styling, for example)
  • Fancy CSS selectors with evergreen support
  • Empty elements used for spacing (as long as they don't have semantic meaning)
  • Nesting elements where this helps with semantic meaning (for example blockquotes)
  • Abusing attributes such as aria-roledescription as selectors

The title of this project is a sly reference.
